นการพัฒนาเว็บแอปพลิเคชันด้วย Node.js (Express.js) ร่วมกับฐานข้อมูล MySQL โดยใช้เทมเพลตเอนจิน EJS ในการแสดงผลหน้าเว็บครับ โค้ดถูกออกแบบมาให้รองรับผู้ใช้งาน 2 ระดับหลักๆ คือ นักศึกษา (Student) และ ผู้ดูแลระบบ (Admin) นี่คือคำอธิบายการทำงานของระบบหลักๆ และการดึง Database View (v_...) ในแต่ละหน้าครับ

ภาพรวมการทำงานหลักของระบบ (Core System)
การเชื่อมต่อฐานข้อมูล: ใช้ mysql2/promise สร้าง Connection Pool เพื่อจัดการการเชื่อมต่อกับ MySQL แบบ Asynchronous ทำให้ระบบไม่บล็อกการทำงานเวลาดึงข้อมูล

ระบบ Session (Authentication): ใช้ express-session ในการจำสถานะการล็อกอิน เมื่อผู้ใช้ล็อกอินสำเร็จ ข้อมูลจะถูกเก็บไว้ใน req.session.user

Middleware (requireLogin): เป็นด่านตรวจความปลอดภัย ก่อนจะเข้าหน้าเว็บใดๆ (เช่น /lms, /events) ระบบจะเช็คก่อนว่ามี Session หรือยัง ถ้ายังจะถูกเด้งกลับไปหน้า /login ทันที

อธิบายการทำงานและการดึง View ในแต่ละหน้า (Routes & Views)
1. หน้า Dashboard (/) - หน้าหลักสรุปข้อมูล
การทำงาน: เป็นหน้าแรกที่ผู้ใช้เห็นหลังล็อกอิน โดยจะตรวจสอบสิทธิ์ว่าเป็น Admin หรือ Student ถ้าเป็น Admin จะดึงข้อมูลภาพรวมของทุกคน (หรือค่าเฉลี่ย) แต่ถ้าเป็น Student จะดึงเฉพาะข้อมูลที่มี UserID ตรงกับตัวเอง

Views ที่เรียกใช้:

v_student_gpa: ดึงเกรดเฉลี่ย (GPAX)

student_daily_schedule: ดึงตารางเรียนรายวัน

student_pending_assignments: ดึงงานที่ค้างส่ง

v_student_dashboard: ดึงข้อมูลสรุปสำหรับแสดงใน Dashboard

v_student_credits: ดึงหน่วยกิตรวมที่ลงทะเบียนไปแล้ว

v_subject_workload: ดึงสรุปภาระงานในแต่ละรายวิชา

v_teacher_reviews: ดึงคะแนนรีวิวของอาจารย์

v_faculty_stats: ดึงสถิติจำนวนนักศึกษาแยกตามคณะ

2. หน้า LMS (/lms) - ระบบการบ้านและเกรด
การทำงาน: แสดงรายการการบ้าน วันกำหนดส่ง สถานะการส่งงาน และผลการเรียน (เกรด) โดยแยกมุมมองระหว่าง Admin (เห็นการบ้านทั้งหมด) และนักศึกษา (เห็นเฉพาะงานของตัวเอง)

Views ที่เรียกใช้:

v_student_lms_assignments: (สำหรับ Student) ดึงรายการการบ้านและสถานะการส่งงาน

v_student_grades: ดึงเกรดของรายวิชาต่างๆ

หมายเหตุ: สำหรับ Admin จะไม่ได้ใช้ View ในส่วนของการบ้าน แต่ใช้คำสั่ง JOIN สดระหว่างตาราง StdAssignment, Subject, และ AssignScore แทน

3. หน้า Registration (/registration) - ระบบลงทะเบียนเรียน
การทำงาน: แสดงรายวิชาที่เปิดให้ลงทะเบียน (ที่ยังไม่ได้ลง) และรายวิชาที่นักศึกษาลงทะเบียนไปแล้ว พร้อมคำนวณผลรวมหน่วยกิตที่ลงไปแล้วทั้งหมด

Views ที่เรียกใช้:

v_course_details: ดึงรายละเอียดรายวิชาและตารางสอน (กรองเฉพาะวิชาที่นักศึกษายังไม่ได้ลงทะเบียน)

v_student_registered_courses: ดึงรายวิชาที่นักศึกษาคนนี้ลงทะเบียนเรียนไปแล้ว

v_subject_workload: ดึงข้อมูลภาระงาน/การบ้านของแต่ละวิชา เพื่อช่วยตัดสินใจก่อนลงทะเบียน

4. หน้า Events (/events) - ระบบกิจกรรม
การทำงาน: แสดงรายการกิจกรรมทั้งหมด โดยแบ่งเป็น "กิจกรรมที่กำลังจะมาถึง (ยังไม่ได้ลงทะเบียน)" และ "กิจกรรมที่ลงทะเบียนไปแล้ว"

Views ที่เรียกใช้:

v_event_attendees: ดึงข้อมูลผู้เข้าร่วมกิจกรรมทั้งหมด (รายชื่อ/จำนวน)

หมายเหตุ: รายการกิจกรรมหลักดึงผ่านการ LEFT JOIN ระหว่างตาราง Event และ Register

5. หน้า Academic (/academic) - ตารางสอนและหลักสูตร
การทำงาน: แสดงข้อมูลคณะ/สาขา และตารางเรียน โดย Admin จะเห็นตารางสอนรวมของทุกวิชา แต่นักศึกษาจะเห็นเฉพาะตารางเรียนของวิชาที่ตัวเองลงทะเบียนไว้

Views ที่เรียกใช้:

v_course_details: (สำหรับ Admin) ดูตารางสอนรวมทั้งหมด

student_daily_schedule: (สำหรับ Student) ดูตารางเรียนของตัวเอง

6. หน้า Evaluation (/evaluation) - ระบบประเมินอาจารย์ผู้สอน
การทำงาน: แสดงรายการงานที่ส่งแล้วแต่ยังไม่ได้ประเมินอาจารย์ (Pending Reviews), ประวัติการประเมินที่ทำไปแล้ว, และคะแนนรวมของอาจารย์แต่ละท่าน

Views ที่เรียกใช้:

v_student_completed_reviews: ดึงประวัติการรีวิวที่นักศึกษาคนนี้เคยประเมินไปแล้ว

v_teacher_reviews: ดึงข้อมูลเรตติ้งเฉลี่ยของอาจารย์ทั้งหมด

หมายเหตุ: รายการที่รอประเมิน (Pending) จะใช้คำสั่ง JOIN ที่ค่อนข้างซับซ้อนเพื่อเช็คงานที่ส่งแล้ว (Status = 'SUBMITTED') และยังไม่มีข้อมูลในตาราง ReviewTeacher

7. ระบบ User & Settings (/login, /register, /settings)
การทำงาน: จัดการการเข้าสู่ระบบ สมัครสมาชิก (เซ็ตค่าเริ่มต้นเป็น UserType = "STD") และการแก้ไขข้อมูลส่วนตัว (ชื่อ-นามสกุล, รหัสผ่าน)

Views ที่เรียกใช้:

v_user_profile: (ในหน้า /settings) ดึงข้อมูลโปรไฟล์ของผู้ใช้ปัจจุบันมาแสดง
